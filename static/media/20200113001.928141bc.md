### Windows 10 시스템에서 github 계정 변경 후 커밋진행시 오류발생

### 발생오류 => permission denied.

### 원인

```note
신규 프로젝트를 생성후 github계정을 바꿔서 commit 진행시
이전 gihub 아이디가 남아있어서 permission denied 오류 발생
```

### 해결방법

1. 왼쪽하단 검색창에서 "자격 증명 관리자" 입력 후 검색한다.
   ![](../../../images/20200113001_1.png)
2. "Windows 자격증명" 클릭한다.
   ![](../../../images/20200113001_2.png)
3. "git:https://github.com" 클릭 후 "편집"을 클릭한다.
   ![](../../../images/20200113001_3.png)
4. 일반 자격 증명 편집 화면에서 사용자이름, 암호 변경 후 저장한다.
   ![](../../../images/20200113001_4.png)

> 위와 같이 하면 조금은 번거롭지만 Github 계정을 변경 하면서 commit이 가능하다.
